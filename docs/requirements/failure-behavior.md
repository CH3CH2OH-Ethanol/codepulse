# CodePulse 失败行为

## 文档目的

本文档描述第一版 CodePulse 在网络中断、更新乱序、文件被拒绝和教师服务端重启时的用户可观察行为。它规定学生插件与教师端应呈现的状态，不决定 HTTP 状态码、重试算法、存储格式或具体类设计。

## 基本原则

- 请求没有响应只表示课堂状态未知，不能据此判断课堂不存在或已经结束。
- 学生本地编辑、运行和调试不依赖 CodePulse 网络连接。
- 服务端只使用已经接受的更新改变课堂状态；失败或过期的更新不得破坏此前有效状态。
- 插件不因重试而补发完整编辑历史，只恢复当前需要同步的最新状态。
- 学生端可以显示详细的本地错误；教师端只显示完成课堂协助所需且不会泄露被拒绝文件信息的错误类别。
- 第一版不承诺在教师服务端重启后恢复内存课堂。

## 学生端状态

| 状态 | 含义 | 是否发送代码或活动信息 |
| --- | --- | --- |
| 未加入 | 学生没有有效的课堂身份 | 否 |
| 正在加入 | 学生已经确认授权，插件正在请求加入课堂 | 否 |
| 同步中 | 服务端确认课堂有效，插件可以发送心跳和允许同步的数据 | 是 |
| 正在重连 | 插件暂时无法确认服务端和课堂状态 | 不发送普通更新，只尝试恢复连接 |
| 已离开 | 学生主动停止本次同步 | 否 |
| 课堂不可用 | 服务端明确表示原课堂不存在、无法恢复或加入凭证失效 | 否 |
| 课堂已结束 | 插件收到教师结束课堂的明确通知 | 否 |

学生端状态描述插件掌握的事实。“正在重连”不表示服务端已经认定学生离线；教师端的“离线”是服务端根据心跳超时作出的故障检测结果。

## 暂时断线与自动重连

### 进入重连状态

插件在发送心跳、代码快照或其他同步请求时，如果没有收到服务端响应或无法建立连接，必须：

1. 将学生端状态改为“正在重连”；
2. 明确说明课堂状态暂时无法确认；
3. 保持学生本地编辑、运行和调试不受影响；
4. 自动尝试恢复连接；
5. 允许学生在重连期间主动离开并立即停止后续同步。

一次文件规则拒绝或其他明确的请求错误不等于连接中断。服务端仍然能够响应时，插件应保持“同步中”，并单独显示该次同步错误。

具体重试间隔和退避算法在实现前决定。重试必须限制频率，不能通过高频请求持续占用教师服务端或课堂网络。

### 教师端离线显示

服务端初始每十秒应收到一次学生心跳。连续三十秒未收到心跳后，教师端将该学生标记为“离线”。

学生离线后，教师端必须：

- 保留并允许查看最后一次成功同步的代码；
- 显示最后心跳时间和最后代码同步时间；
- 明确提示现有代码可能已经过期；
- 不把“离线”解释为学生已经退出、关闭电脑或停止编程。

心跳和离线阈值是初始目标，后续通过课堂网络与负载实验验证。

### 恢复连接

插件重新联系到服务端后，根据服务端的明确结果处理：

- 原课堂仍然有效且参与者身份有效：恢复“同步中”，继续发送心跳，并发送当前活动文件的最新完整快照；
- 原课堂不可恢复或参与者凭证失效：停止重试，显示“课堂不可用”，要求学生使用新的加入 URL 重新加入；
- 服务端明确通知课堂已经结束：停止重试，显示“课堂已结束”。

如果服务端在课堂结束后已经删除全部会话状态，它可能无法区分“曾经结束的课堂”和“不存在的课堂”。只有服务端确实掌握结束原因时，插件才显示“课堂已结束”；其他明确不可恢复的情况统一显示“课堂不可用”。

## 过期与重复更新

每位参与者的每个同步文件都必须拥有单调递增的快照版本。版本只用于比较同一课堂、同一参与者和同一文件的更新顺序，不能跨这些边界比较。

服务端为每个已经同步的文件记录当前接受的最高版本：

- 收到更高版本时，接受快照并更新该文件的最新状态；
- 收到更低版本时，将其识别为过期更新，不允许覆盖较新状态；
- 收到相同版本时，将其识别为重复请求，幂等确认而不重复改变状态。

客户端时间戳和服务端接收顺序不能代替快照版本。客户端时间可能发生调整，而网络延迟可能使较早发送的请求更晚到达。

请求发送后没有收到响应时，插件可以使用相同版本重试。学生继续修改文件后产生更高版本；较旧请求随后到达时，服务端仍必须保留较新版本。

## 不允许同步的文件

### 插件本地拒绝

插件必须在文件内容离开学生电脑前执行工作区、符号链接、敏感路径、扩展名、文本类型和大小检查。

文件被本地拒绝时：

- 不发送文件内容；
- 不发送文件路径；
- 学生端显示工作区内相对路径和详细拒绝原因；
- 插件可以向教师服务端发送不含路径和内容的同步错误事件；
- 错误事件只包含安全化错误类别、发生时间和识别学生所需的课堂身份；
- 插件保持“同步中”，其他合法文件继续正常同步。

安全化错误类别可以包括“文件类型不允许”“文件超过大小限制”“文件被安全规则阻止”和“文件路径无效”。加入课堂前必须告知学生：同步错误类别和发生时间会提供给教师。

### 服务端拒绝

服务端不能信任客户端已经正确执行全部检查。它必须再次执行自己能够验证的规则，例如扩展名、相对路径、敏感路径和请求大小检查。服务端无法根据网络请求可靠判断学生本地文件是否为符号链接，因此客户端检查仍然不可替代。

服务端拒绝更新时：

- 不保存被拒绝的文件内容；
- 返回可供学生理解且不暴露服务端内部细节的错误类别；
- 教师端可以显示安全化错误类别和发生时间；
- 不向教师显示被拒绝文件的路径或内容；
- 不删除或覆盖该学生此前成功同步的合法快照；
- 不因一次文件拒绝把整个学生连接标记为离线。

普通编辑活动时间与代码快照是否被接受分别处理。学生编辑被拒绝的文件时，教师可以看到仍有编辑器活动，但不能据此得知被拒绝文件的名称、路径或内容。

## 学生主动离开时的通知失败

学生主动离开必须首先在本地生效。插件立即停止读取和发送新的代码、活动信息与心跳，并停止自动重连；它不等待服务端确认后才保护学生的退出选择。

插件可以尽力发送一次离开通知，然后清除本次课堂凭证：

- 服务端收到通知时，教师端显示“已离开”；
- 通知因断网而未送达时，服务端无法知道学生主动离开，最终只能在心跳超时后显示“离线”；
- 插件不得为了保证教师看到“已离开”而在学生退出后继续后台重连。

因此，“已离开”表示服务端确实收到主动离开通知；学生端停止同步不保证教师端一定能够观察到该原因。

## 教师服务端重启

第一版教师服务端是课堂状态的单一权威节点，课堂数据默认只保存在该进程的内存中。服务端重启会丢失正在进行的课堂会话、参与者状态、已同步代码和未导出的最终快照。

服务端停止期间：

- 学生插件显示“正在重连”，因为它无法判断故障原因；
- 教师 Dashboard 无法获取课堂状态；
- 学生继续在本地编辑、运行和调试。

服务端重新启动后：

1. 教师端显示当前没有正在进行的课堂；
2. 原学生插件联系服务端后得到原课堂不可用的明确结果；
3. 插件停止自动重试并显示“课堂不可用”；
4. 教师需要创建新课堂并发送新的加入 URL；
5. 学生重新查看授权范围并确认加入；
6. 系统不宣称能够恢复重启前仅存在于内存中的课堂数据。

教师服务端重启不等同于教师主动结束课堂。第一版不通过服务端集群提供高可用。以后可以通过教师电脑本地持久化改善进程重启恢复，但这与多节点复制和故障转移是不同问题。

## 开放问题

以下问题留待实现或后续架构决策：

- 自动重试采用何种退避算法和最大间隔；
- 快照版本如何生成、保存以及在插件重启后恢复；
- 同步错误事件如何合并或限流，避免重复提示；
- 服务端是否短期保留不含学生代码的课堂终止标记；
- 是否以及何时为教师服务端加入本地持久化；
- HTTP 状态码和机器可读错误代码的具体设计。
