# CodePulse User Stories

## Scope

这些用户故事只描述第一版局域网课堂场景。学生使用 VS Code 主动加入课堂并选择同步工作区；系统提供可观察的事实，由教师判断学生是否需要帮助。

## Teacher Stories

### T1：确定关注顺序

作为教师，我希望看到已加入学生的在线状态、最近代码修改时间和最近编辑活动时间，从而优先关注可能长时间没有进展的学生。

### T2：了解学生当前的代码上下文

作为教师，我希望在决定帮助某位学生时查看其当前活动文件的最新代码，从而在不打断学生操作的情况下了解问题上下文并给出更有针对性的提示。

## Student Stories

### S1：减少求助时的操作

作为学生，我希望在主动加入课堂并选择同步工作区后，让教师看到我当前正在编辑的代码，从而在需要帮助时减少反复说明或共享屏幕的操作。

### S2：掌握代码同步状态

作为学生，我希望随时看到代码同步是否正在进行，并能主动离开课堂以停止同步，从而清楚掌握自己的代码何时处于课堂观察范围内。
