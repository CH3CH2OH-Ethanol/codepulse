# CodePulse 课堂生命周期

## 文档目的

本文档描述第一版 CodePulse 课堂会话从创建到结束的用户可观察流程。它规定参与者能够看到和执行的行为，不决定 URL 协议、网络接口或框架实现。跨文档术语见 [CodePulse 需求术语表](glossary.md)。

## 核心概念

### 课堂会话

课堂会话是教师本次启动并最终结束的临时空间，不代表跨多次课堂长期存在的课程。教师输入的课堂名称用于帮助学生确认是否进入了正确的课堂；系统生成的唯一会话标识用于区分不同课堂。

### 学生显示名称与参与者标识

学生输入的名称只用于课堂展示，可以与其他学生重复。系统必须为每位学生分配本次课堂内唯一且稳定的参与者标识，并在教师界面中自动区分重名学生。

## 参与者状态

| 状态 | 含义 | 是否自动恢复同步 |
| --- | --- | --- |
| 同步中 | 学生已确认加入，插件正在发送心跳和允许同步的数据 | 不适用 |
| 离线 | 服务端在超时阈值内未收到学生心跳，可能是暂时网络故障 | 是 |
| 已离开 | 服务端已经收到学生主动离开通知 | 否 |
| 课堂已结束 | 教师已经关闭整个课堂会话 | 否 |

## 主流程

### 1. 教师创建课堂

1. 教师启动本地 CodePulse。
2. 教师输入本次课堂的名称。
3. 教师配置本次课堂允许同步的文件扩展名。
4. 教师确认创建课堂。
5. 系统创建唯一的课堂会话，并生成本次课堂专用的加入 URL；加入凭证包含至少 128 位来自密码学安全随机源的随机性。
6. 教师将加入 URL 提供给学生。

课堂名称只用于人工确认，不能作为课堂的唯一身份或访问凭证。加入凭证不能根据课堂名称、创建时间或递增序号等可预测信息生成；加入 URL 不包含课堂名称、学生信息、本地文件路径或其他非必要敏感信息。加入凭证只能用于对应课堂，并在课堂结束后失效。

第一版不允许教师在课堂创建后静默扩大文件扩展名允许列表。详细的工作区、文件类型和固定安全规则见 [CodePulse 文件同步边界](file-sync-boundary.md)。

### 2. 学生确认加入

1. 学生通过 VS Code 插件打开加入 URL。
2. 插件显示课堂名称，帮助学生确认目标课堂。
3. 插件说明同步的数据类型、接收方、持续时间和停止方式，包括未保存编辑器内容可能同步，以及安全化错误类别和发生时间可能提供给教师。
4. 学生输入课堂显示名称。
5. 学生选择本次课堂允许同步的工作区。
6. 插件显示课堂允许的文件类型和同步边界。
7. 学生明确确认加入。
8. 系统分配唯一参与者标识，并在发生重名时生成稳定的可区分标签。
9. 插件开始同步，并持续显示正在同步的状态。
10. 如果加入时活动文件位于所选工作区并通过全部同步检查，插件发送该文件的完整初始快照；没有符合条件的活动文件时，教师端显示“暂无已同步代码”。

学生确认前不得为 CodePulse 读取或发送工作区代码，也不得发送编辑活动信息。一个课堂最多接受三十个参与者记录；同步中、离线和已离开记录都计入限制，第 31 个加入请求必须被拒绝并显示“课堂人数已满”。

### 3. 课堂中持续同步

1. 插件只观察学生选择的工作区。
2. 插件只同步符合课堂允许类型的加入时活动文件，以及课堂期间实际发生文本修改的文件；仅切换或查看文件不会触发同步。
3. 加入课堂时不得自动上传整个工作区。
4. 插件定期发送心跳，使服务端能够判断连接状态。
5. 教师先查看全班学生状态，再选择一名学生查看其当前同步文件的最新已接受代码。

第一版不同步普通终端内容、运行结果、工作区外文件或完整编辑历史。

在教师 Dashboard 已打开、服务端与 Dashboard 连接正常的条件下，从服务端接受代码快照到 Dashboard 显示该快照，正常目标为约两秒，第一版可接受上限为三秒。该时间不包含学生停止输入前的编辑过程，也不使用学生设备时间计算；具体轮询间隔和测试负载在实现阶段确定。

### 4. 处理重名学生

1. 系统允许多名学生使用相同显示名称。
2. 教师界面必须为重名学生显示稳定且可区分的标签。
3. 学生界面必须显示自己最终呈现给教师的标签。
4. 其他学生离开课堂时，已有学生的标签不得重新编号。

### 5. 暂时断线与自动恢复

1. 插件初始每十秒发送一次心跳。
2. 服务端连续三十秒未收到心跳后，将学生显示为离线。
3. 短暂断线期间，学生继续在本地编辑和运行代码。
4. 网络恢复且原课堂仍有效时，插件使用原参与者身份自动恢复同步，不要求学生重新输入名称或选择工作区。
5. 断线期间，插件为每个发生文本修改且通过同步检查的文件保留一个最新待发送完整快照；同一文件的新快照替换旧快照。
6. 恢复后，插件发送所有待发送文件的最新完整快照，并只在服务端确认后移除对应待发送状态。
7. 插件不补发断线期间的每一次编辑操作或完整编辑历史。

心跳和离线阈值是初始目标，后续必须通过课堂负载与网络实验验证。

### 6. 插件重启后的恢复

1. 插件重启后如果发现未结束的本地课堂记录，必须显示原课堂名称、工作区和同步范围。
2. 插件必须要求学生明确确认是否恢复；确认前不得读取或发送代码、编辑活动或心跳。
3. 学生确认且服务端确认原课堂有效时，插件使用原参与者身份恢复同步。
4. 学生拒绝恢复时，插件按主动离开处理，尽力发送一次离开通知并清除本地课堂记录。

### 7. 学生主动离开

1. 学生主动选择离开课堂。
2. 插件立即停止读取和发送新的代码、活动信息与心跳，并停止自动重连；本地退出不等待服务端确认。
3. 插件尽力发送一次离开通知，然后清除本次加入凭证。
4. 服务端收到通知时，教师端将该学生显示为已离开；通知因断网而未送达时，教师端最终只能在心跳超时后显示离线。
5. 教师端保留学生离开前最后收到的代码，直到本次课堂结束。
6. 学生再次加入时必须重新确认并选择工作区，不能自动恢复同步。

加入前必须明确告知学生：主动离开会停止新的同步，但此前已经同步的代码会保留到本次课堂结束。

“已离开”只表示服务端确实收到主动离开通知。系统不得为了保证通知送达而在学生退出后继续后台重连。

### 8. 教师结束课堂

教师选择结束课堂时，系统提供以下选项：

- 导出最终快照并结束；
- 直接结束并删除；
- 取消。

教师确认“导出最终快照并结束”或“直接结束并删除”后：

1. 系统将课堂标记为已结束，不再接受学生更新。
2. 所有仍连接并收到结束通知的学生端显示“课堂已结束，代码同步已停止”。断线学生如果之后只能确认原会话不可恢复，则显示“课堂不可用”。
3. 选择导出时，系统根据结束时的最终状态生成 ZIP，并通过浏览器下载流程提供给教师。
4. ZIP 生成或下载开始后，系统继续保留内存课堂数据并允许再次下载。
5. 教师明确确认已经保存导出文件后，系统清除内存中的代码、参与者状态和活动数据。
6. 选择直接结束时，系统不生成文件并清除内存中的代码、参与者状态和活动数据。
7. 系统保留不含学生代码和身份信息的课堂终止标记，最长二十四小时或直到服务端进程停止，以先发生者为准。

如果 ZIP 生成失败，或教师没有确认已经保存导出文件，系统不得自动删除内存数据。教师可以重新生成或下载 ZIP，也可以明确选择放弃导出并删除课堂数据。课堂在等待处理期间保持已结束状态并拒绝新的学生更新。

## 最终快照边界

第一版最终快照是 ZIP 文件。ZIP 名称由教师指定；最终保存位置由浏览器和操作系统控制。ZIP 内为每位参与者建立由安全化展示标签和短参与者标识组成的独立目录，并保留文件的工作区相对目录结构。

最终快照只包括：

- 课堂期间实际同步过的文件；
- 每个文件最后收到的版本；
- 用于区分学生和文件相对位置的必要信息。

最终快照不包括：

- 完整编辑历史或过程回放；
- 学生本地未同步的文件；
- 普通终端内容或运行结果；
- 课堂结束后的任何变化。

导出最终快照是教师明确选择将课堂代码写入本地磁盘的操作。第一版导出文件不加密；教师确认导出后，文件保存和访问权限由教师电脑及操作系统负责。默认情况下，课堂代码和参与者状态只保存在教师服务端内存中。

## 开放问题

以下问题尚未决定，不在本文档中预先选择实现：

- 加入 URL 的具体协议和打开方式；
- 重名标签和导出目录短参与者标识的具体格式；
- 学生加入时如何展示文件类型和敏感文件边界；
- 导出文件名的安全化规则；
- 心跳、离线判断和代码显示延迟目标的测试负载与调整条件。
