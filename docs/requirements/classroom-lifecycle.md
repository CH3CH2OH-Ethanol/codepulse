# CodePulse 课堂生命周期

## 文档目的

本文档描述第一版 CodePulse 课堂会话从创建到结束的用户可观察流程。它规定参与者能够看到和执行的行为，不决定 URL 协议、网络接口或框架实现。

## 核心概念

### 课堂会话

课堂会话是教师本次启动并最终结束的临时空间，不代表跨多次课堂长期存在的课程。教师输入的课堂名称用于帮助学生确认是否进入了正确的课堂；系统生成的唯一会话标识用于区分不同课堂。

### 学生显示名称与参与者标识

学生输入的名称只用于课堂展示，可以与其他学生重复。系统必须为每位学生分配本次课堂内唯一且稳定的参与者标识，并在教师界面中自动区分重名学生。

## 参与者状态

| 状态 | 含义 | 是否自动恢复同步 |
| --- | --- | --- |
| 同步中 | 学生已确认加入，插件正在发送心跳和允许同步的数据 | 不适用 |
| 离线 | 服务端在超时阈值内未收到学生心跳，可能是暂时网络故障 | 是 |
| 已离开 | 学生主动结束本次同步 | 否 |
| 课堂已结束 | 教师已经关闭整个课堂会话 | 否 |

## 主流程

### 1. 教师创建课堂

1. 教师启动本地 CodePulse。
2. 教师输入本次课堂的名称。
3. 教师配置本次课堂允许同步的文件扩展名。
4. 教师确认创建课堂。
5. 系统创建唯一的课堂会话，并生成本次课堂专用且不易猜测的加入 URL。
6. 教师将加入 URL 提供给学生。

课堂名称只用于人工确认，不能作为课堂的唯一身份或访问凭证。

第一版不允许教师在课堂创建后静默扩大文件扩展名允许列表。详细的工作区、文件类型和固定安全规则见 [CodePulse 文件同步边界](file-sync-boundary.md)。

### 2. 学生确认加入

1. 学生通过 VS Code 插件打开加入 URL。
2. 插件显示课堂名称，帮助学生确认目标课堂。
3. 插件说明同步的数据范围、接收方、同步持续时间和停止方式。
4. 学生输入课堂显示名称。
5. 学生选择本次课堂允许同步的工作区。
6. 插件显示课堂允许的文件类型和同步边界。
7. 学生明确确认加入。
8. 系统分配唯一参与者标识，并在发生重名时生成稳定的可区分标签。
9. 插件开始同步，并持续显示正在同步的状态。

学生确认前不得发送工作区代码或编辑活动信息。

### 3. 课堂中持续同步

1. 插件只观察学生选择的工作区。
2. 插件只同步符合课堂允许类型并在课堂期间实际打开或修改的文件。
3. 加入课堂时不得自动上传整个工作区。
4. 插件定期发送心跳，使服务端能够判断连接状态。
5. 教师先查看全班学生状态，再选择一名学生查看其当前活动文件的最新代码。

第一版不同步普通终端内容、运行结果、工作区外文件或完整编辑历史。

### 4. 处理重名学生

1. 系统允许多名学生使用相同显示名称。
2. 教师界面必须为重名学生显示稳定且可区分的标签。
3. 学生界面必须显示自己最终呈现给教师的标签。
4. 其他学生离开课堂时，已有学生的标签不得重新编号。

### 5. 暂时断线与自动恢复

1. 插件初始每十秒发送一次心跳。
2. 服务端连续三十秒未收到心跳后，将学生显示为离线。
3. 短暂断线期间，学生继续在本地编辑和运行代码。
4. 网络恢复且原课堂仍有效时，插件使用原参与者身份自动恢复同步，不要求学生重新输入名称或选择工作区。
5. 恢复后，插件发送当前活动文件的最新完整快照，以修复断线期间遗漏的更新。
6. 插件不补发断线期间的每一次编辑操作。

心跳和离线阈值是初始目标，后续必须通过课堂负载与网络实验验证。

### 6. 学生主动离开

1. 学生主动选择离开课堂。
2. 插件立即停止心跳、重连和代码发送，并清除本次加入凭证。
3. 教师端将该学生显示为已离开，而不是离线。
4. 教师端保留学生离开前最后收到的代码，直到本次课堂结束。
5. 学生再次加入时必须重新确认并选择工作区，不能自动恢复同步。

加入前必须明确告知学生：主动离开会停止新的同步，但此前已经同步的代码会保留到本次课堂结束。

### 7. 教师结束课堂

教师选择结束课堂时，系统提供以下选项：

- 导出最终快照并结束；
- 直接结束并删除；
- 取消。

教师确认结束后：

1. 系统将课堂标记为已结束，不再接受学生更新。
2. 所有仍连接的学生端显示“课堂已结束，代码同步已停止”。
3. 选择导出时，系统根据结束时的最终状态生成快照。
4. 导出成功后，系统清除内存中的课堂数据。
5. 选择直接结束时，系统不生成文件并清除内存中的课堂数据。

如果导出失败，系统不得自动删除内存数据。教师可以重试导出，或明确选择放弃导出并删除课堂数据。

## 最终快照边界

最终快照只包括：

- 课堂期间实际同步过的文件；
- 每个文件最后收到的版本；
- 用于区分学生和文件相对位置的必要信息。

最终快照不包括：

- 完整编辑历史或过程回放；
- 学生本地未同步的文件；
- 普通终端内容或运行结果；
- 课堂结束后的任何变化。

导出最终快照是教师明确选择将课堂代码写入本地磁盘的操作。默认情况下，课堂数据只在教师端内存中保留到课堂结束。

## 开放问题

以下问题尚未决定，不在本文档中预先选择实现：

- 加入 URL 的具体协议和打开方式；
- 重名标签的具体展示格式；
- 学生加入时如何展示文件类型和敏感文件边界；
- 最终快照的目录结构和文件命名；
- 心跳、离线判断和代码显示延迟的最终参数。
