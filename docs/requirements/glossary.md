# CodePulse 需求术语表

## 文档目的

本文档为第一版 CodePulse 的需求、架构和验收文档提供统一术语。跨文档使用这些术语时必须遵循这里的定义；如果定义需要改变，应同时检查所有引用位置，不能在单个文档中悄悄改变含义。

## 课堂与身份

### 课堂会话

教师本次创建并最终结束的临时课堂空间。课堂会话不代表跨多次课堂长期存在的课程。

### 课堂名称

教师为课堂输入、供学生人工确认目标课堂的展示文本。课堂名称不是课堂会话标识，也不是访问凭证。

### 课堂会话标识

系统为每个课堂会话分配的唯一标识。它用于区分课堂，不因课堂名称相同而冲突，也不必承担保密或授权作用。

### 加入凭证

包含至少 128 位密码学安全随机性的秘密值，只授予持有者加入对应课堂的能力。加入凭证不能授予教师权限，并在课堂结束后失效。

### 参与者凭证

学生成功加入后获得的不透明秘密值，只授予持有者以对应参与者身份提交代码快照、编辑活动、心跳和离开通知的能力。参与者凭证不能读取课堂数据、代表其他参与者提交或执行教师操作，并在参与者离开或课堂结束后失效。

### 教师会话

教师在本机建立的服务端会话，只授予读取全班状态与学生代码、结束课堂和导出最终快照等教师能力。教师会话不能通过学生加入 URL 建立，也不能依赖前端隐藏按钮代替服务端授权检查。

### Bearer credential

持有者出示后即可使用其权限的凭证，类似不记名票据。加入凭证、参与者凭证和教师会话标识都必须按秘密处理；泄露后，获得者可能在凭证失效前冒充对应身份。

### 参与者标识

学生成功加入后，系统为其分配的课堂会话内唯一且稳定的内部标识。学生离开后再次加入会形成新的参与者记录和标识。

### 学生显示名称

学生加入课堂时输入的展示名称。不同学生可以输入相同的显示名称。

### 展示标签

教师和对应学生能够看到的稳定标签。出现重复显示名称时，系统使用展示标签区分相关参与者；其他学生离开不能导致已有标签重新编号。

## 工作区与文件

### 所选工作区

学生为本次课堂明确选择的一个 VS Code 工作区文件夹。同步授权只覆盖该文件夹内且通过全部同步检查的文件。

### 加入时活动文件

学生确认加入时，VS Code 当前获得焦点的文本文件。该文件只有位于所选工作区并通过全部同步检查时，才会作为初始完整快照发送。

### 当前同步文件

教师服务端最近成功接受代码快照的文件。学生仅切换或查看另一个文件不会改变当前同步文件；该文件产生文本修改并成功同步后，才成为新的当前同步文件。

### 文本修改

VS Code 编辑器中的文档内容发生变化，包括尚未保存到磁盘的变化。移动光标、选择文本和切换文件不属于文本修改。

### 代码快照

某个允许同步文件在一个时刻的完整编辑器文本内容及其工作区相对路径和版本。第一版不发送字符级差异。

### 快照版本

用于比较同一课堂会话、同一参与者和同一文件更新顺序的单调递增值。它不能跨课堂、参与者或文件比较。

### 待发送快照

尚未得到服务端成功确认的文件最新完整快照。同一文件再次修改时，新快照替换该文件此前的待发送快照；它不是编辑历史。

## 时间与活动

Dashboard 中用于比较参与者的权威时间均由教师服务端记录。第一版不计算学生设备与教师设备之间的时钟偏移。

### 代码同步时间

教师服务端最近一次成功接受参与者代码快照的时间。它不宣称是学生实际键入内容的精确时间。

### 编辑活动时间

教师服务端最近一次收到参与者文本修改事件的时间。允许同步文件和被拒绝文件中的文本修改都可以更新该时间，但被拒绝文件的名称、路径和内容不能随事件发送。

### 最后心跳时间

教师服务端最近一次成功收到参与者心跳的时间。

### 不活跃基准时间

参与者存在编辑活动时，使用最近编辑活动时间；参与者尚无编辑活动时，使用成功加入时间。

### 不活跃提醒

不活跃基准时间距当前服务端时间超过教师配置阈值时显示的注意提示。它只表示近期没有观察到文本修改，不表示系统已经判断学生卡住、离开或停止编程。

## 状态与故障

### 同步中

服务端确认课堂和参与者身份有效，插件可以发送心跳及允许同步的数据。

### 正在重连

插件无法联系服务端或没有收到响应，因而无法确认课堂状态。该状态表示未知，不等于课堂不存在或已结束。

### 离线

服务端在离线阈值内没有收到参与者心跳后的故障检测结果。它不证明学生主动离开、关闭电脑或停止编程。

### 已离开

服务端已经成功收到学生主动离开通知。学生本地停止同步但通知未送达时，教师端只能观察到离线。

### 课堂不可用

服务端明确表示原课堂不存在、无法恢复或加入凭证失效。插件收到该结果后停止自动重试。

### 课堂已结束

插件或 Dashboard 收到教师已结束课堂的明确结果。只有服务端确实掌握结束原因时才能使用该状态。

### 安全化错误类别

不包含被拒绝文件路径、名称或内容的错误分类，例如“文件类型不允许”“文件超过大小限制”“文件被安全规则阻止”或“文件路径无效”。

### 课堂终止标记

课堂结束后短期保留的最小状态，只表示课堂曾存在、已经结束及结束时间。它不包含学生代码、身份信息、文件路径或活动记录。

## 导出

### 最终快照

教师明确选择结束并导出时生成的 ZIP 文件。它只包含课堂结束时服务端已接受的每个文件最新版本，以及区分参与者和工作区相对位置所需的信息；不包含完整编辑历史、未同步文件、终端内容或运行结果。
