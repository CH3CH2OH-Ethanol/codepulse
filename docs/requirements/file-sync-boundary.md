# CodePulse 文件同步边界

## 文档目的

本文档规定第一版 CodePulse 在什么条件下允许文件离开学生电脑，以及哪些规则不能由教师或学生关闭。它描述产品和隐私边界，不决定 VS Code API、网络协议或服务端实现。

## 核心原则

- 默认拒绝；文件只有通过全部检查后才能同步。
- 学生只对明确选择的工作区和课堂会话授权。
- 教师配置课堂任务需要的文件类型，但不能放宽系统安全规则。
- 加入课堂时不上传整个工作区，只处理加入时活动文件以及课堂期间实际发生文本修改的文件；仅切换或查看文件不会触发同步。
- 被拒绝的文件必须显示原因，不能静默失败。
- CodePulse 不修改或删除学生本地文件。

## 创建课堂时的教师配置

教师创建课堂前必须配置本次课堂允许同步的文件扩展名，例如 `.java` 或 `.md`。允许列表用于限定课堂任务需要的文件类型，不表示匹配文件一定安全。

课堂创建后，第一版不允许教师在学生不知情的情况下扩大允许列表。如果以后需要在课堂中修改范围，必须重新通知受影响学生并获得确认。

教师不能关闭或覆盖以下固定规则：

- 工作区边界检查；
- 符号链接拒绝；
- 已知敏感文件和目录拒绝；
- 普通文本文件检查；
- 单文件大小限制。

这些配置属于创建课堂用例的一部分，不需要单独建立新的用例。完整流程是：教师输入课堂名称、配置允许扩展名、确认创建，然后系统生成加入 URL。

## 学生授权范围

学生打开加入 URL 后必须：

1. 查看课堂名称和同步说明；
2. 选择一个允许同步的工作区；
3. 查看教师配置的文件扩展名允许列表；
4. 了解编辑器中尚未保存到磁盘的文本内容也可能同步；
5. 了解同步错误类别和发生时间可能提供给教师，但被拒绝文件的路径和内容不会提供给教师；
6. 明确确认加入。

确认前不得读取并发送工作区代码。学生的授权只覆盖本次课堂、所选工作区以及通过全部文件检查的内容。

## 固定检查顺序

插件准备同步文件时必须按以下顺序检查：

1. 当前课堂仍处于同步中状态。
2. 文件来自学生明确选择的工作区。
3. 文件不是符号链接，规范化后的真实路径仍位于工作区内。
4. 文件路径和名称不匹配已知敏感规则。
5. 文件扩展名位于教师配置的允许列表中。
6. 文件是普通文本文件，而不是目录、设备文件或二进制文件。
7. 文件大小不超过 `1 MiB` 的初始上限。
8. 文件是加入时活动文件，或在课堂期间实际发生文本修改，而不是加入时批量扫描或仅切换查看所得。

任何检查失败都必须停止本次同步。后续检查不得覆盖前面检查的拒绝结果。

## 工作区和路径边界

- 插件只能同步所选工作区内的文件。
- 插件发送相对工作区的路径，不发送学生电脑上的绝对路径。
- `..` 路径穿越、工作区外路径和无法安全规范化的路径必须拒绝。
- 第一版拒绝所有符号链接，即使链接目标看起来仍在工作区内。
- 多工作区窗口中，学生必须明确选择一个工作区文件夹作为本次课堂范围。

拒绝所有符号链接会排除少量合法项目结构，但可以避免链接逃逸、链接循环和检查后目标变化等复杂风险。以后只有在出现明确课堂需求时才重新评估。

## 敏感文件和生成内容

系统必须拒绝已知敏感文件和目录。规则至少覆盖以下类别：

- 环境变量和本地凭证文件，例如 `.env` 及其本地变体；
- 私钥、证书和密钥容器，例如常见的私钥文件名以及 `.pem`、`.key`、`.p12`、`.pfx`；
- 版本控制内部数据，例如 `.git/`；
- 依赖和构建输出目录，例如 `node_modules/`、`target/`、`dist/`、`build/`；
- 已知保存访问令牌或本机凭证的工具配置文件。

具体敏感规则需要在实现前单独列出并测试。教师扩展名允许列表不能重新允许这些文件。

## 普通源码中的秘密

第一版不提供通用秘密扫描，也不承诺发现普通源码中手写的 API Key、密码或令牌。通用内容检测容易产生漏报、误报和错误安全感。同步内容来自 VS Code 编辑器文本，因此可能包含尚未保存到磁盘的修改。

加入课堂前必须提醒学生：

- 允许同步的源码仍可能包含自己写入的秘密；
- 学生应检查所选工作区和课堂允许的文件类型；
- 已经同步的内容会保留到本次课堂结束；
- 主动离开只停止后续同步，不自动删除此前内容。

系统仍应阻止能够通过可靠文件名、目录或文件类型规则识别的敏感内容。

## 文件类型和大小

- 第一版只同步普通文本文件。
- 二进制文件即使扩展名位于允许列表中也必须拒绝。
- 单文件大小上限初始为 `1 MiB`。
- 超过上限的文件必须拒绝，并向学生显示文件路径和拒绝原因。
- 大小上限是初始工程目标，后续根据课堂项目和负载实验调整。

## 拒绝反馈

文件被拒绝时，插件必须向学生提供可理解的原因，例如：

- 不在所选工作区；
- 文件类型未被课堂允许；
- 文件属于敏感路径；
- 文件是符号链接或二进制文件；
- 文件超过大小限制。

学生端可以显示工作区内相对路径和详细原因。插件可以向教师端发送不含文件路径和内容的安全化错误类别及发生时间，例如“文件类型不允许”“文件超过大小限制”“文件被安全规则阻止”或“文件路径无效”。

拒绝反馈不得上传被拒绝文件的内容，不得向教师端发送被拒绝文件的路径，也不得泄露工作区外的绝对路径。被拒绝的更新不得删除或覆盖此前成功同步的合法快照。

系统日志不得记录学生源代码、加入凭证、参与者凭证或被拒绝文件路径。

## 分类示例

假设学生选择 `/Users/student/exercise`，教师允许 `.java` 和 `.md`：

| 文件 | 结果 | 原因 |
| --- | --- | --- |
| `src/Main.java` | 允许 | 工作区内的允许文本类型 |
| `README.md` | 允许 | 工作区内的允许文本类型 |
| `notes.txt` | 拒绝 | 扩展名不在允许列表 |
| `.env` | 拒绝 | 已知敏感文件 |
| `.git/config` | 拒绝 | 版本控制内部数据 |
| `target/classes/Main.class` | 拒绝 | 构建输出和二进制文件 |
| `link.java` | 拒绝 | 符号链接，即使名称以 `.java` 结尾 |
| 工作区外的 `Test.java` | 拒绝 | 超出学生授权范围 |
| `src/Huge.java` | 拒绝 | 文件大小为 `20 MiB`，超过初始上限 |

普通 `.java` 文件中可能存在手写秘密；第一版对此提供明确提醒，但不宣称能够自动发现全部秘密。

## 开放问题

以下问题在实现前仍需决定：

- 扩展名允许列表的默认值和大小写规则；
- 文本与二进制文件的具体判断方式；
- 已知敏感文件和目录规则的完整清单；
- 无扩展名源码如何处理；
- 拒绝提示在 VS Code 中的具体展示方式；
- `1 MiB` 上限是否需要根据课堂实验调整。
