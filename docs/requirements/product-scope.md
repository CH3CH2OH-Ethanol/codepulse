# CodePulse 产品范围

## 产品目标

CodePulse 用于课堂编程练习，帮助教师通过学生主动共享的代码状态及时了解学生当前的编程情况，从而判断应优先关注或帮助哪些学生，并为后续讲解提供依据。同时，学生继续使用自己的本地开发环境，并在明确知情的前提下自主控制同步的开始、范围和结束。

## 非目标

### 在线 IDE

CodePulse 不提供代码编辑、运行或调试环境。学生继续使用自己熟悉的本地开发环境，平台只负责同步和展示课堂中明确允许观察的代码状态，不接管学生的开发流程。

### 作业提交平台

CodePulse 不负责收集、归档或批改课后作业。现有教学平台或通信工具已经能够支持一次性文件提交；CodePulse 关注的是课堂编程过程中的持续观察和及时帮助。

### 实时协作编辑器

CodePulse 不允许教师直接修改学生的本地代码。教师可以查看学生主动同步的内容并提供指导，但代码修改仍由学生在自己的开发环境中完成，以保留学生对代码和操作过程的控制。

### 自动评测与自动诊断系统

CodePulse 不在服务端执行学生代码，不自动评分，也不宣称能够判断学生是否真正卡住。系统只提供在线状态、代码变化和编辑活动等可观察事实，由教师结合课堂情境作出判断。

### 课后持续监控

CodePulse 不用于课后持续观察学生。同步只在学生明确加入课堂后进行，并在学生离开课堂后停止；教师不能通过 CodePulse 继续观察已经离开课堂的学生本地活动。

## 访问边界

- 课堂加入凭证只允许持有者加入对应课堂，不能授予教师权限。
- 参与者凭证只允许对应学生提交自己的心跳、活动和代码快照，不能代表其他参与者提交数据。
- 学生不能读取其他学生代码、全班参与者列表或教师专用课堂数据。
- 查看全班状态和学生代码、结束课堂以及导出快照只允许经过教师授权的请求执行。
- 服务端拒绝缺少凭证、凭证无效或权限不足的请求时，不能在错误响应中泄露受保护数据。

具体教师授权机制在架构决策中确定，不在产品范围文档中预先选择。

## 未来候选

### 课堂阶段快照与过程回顾

第一版可选的最终文件快照只保留课堂结束时的最终状态，不提供过程回放。未来可以评估是否在参与者明确知情和同意的前提下保存有限的课堂阶段快照，用于回顾编程过程；这项候选能力不代表实现承诺，是否引入取决于学习价值、访问权限、保存期限、删除机制以及隐私风险的后续评估。

### 直接导出目录

第一版通过浏览器下载 ZIP 最终快照。未来可以评估使用浏览器目录访问能力，让教师授权目标文件夹并直接写入学生目录结构；这项候选能力不代表实现承诺，引入前必须重新评估浏览器兼容性、文件系统授权、失败后的部分文件清理和隐私风险。
