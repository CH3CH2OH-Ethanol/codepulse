# CodePulse 第一版初始验收标准

## 文档状态与用途

状态：M1 Unit 7 待审草案。

本文档使用 Given/When/Then 场景说明如何从系统外部观察第一版 CodePulse
的关键行为。它优先覆盖授权、隐私、数据丢失和故障恢复风险，不为需求目录
中的每个条件机械生成一个场景。

需求规定系统必须做到什么；验收场景提供能够证明这些要求成立的代表性例子；
内部类、接口、状态存储格式和测试框架由后续设计决定。

## 阅读规则

- **Given** 描述场景开始前已经成立的条件。
- **When** 描述触发行为或事件。
- **Then** 描述必须能够观察到的结果。
- **And** 补充与同一结果不可分割的检查。
- “覆盖需求”链接到 [第一版需求目录](requirements-catalog.md)，用于追溯场景来源。

一个场景可以覆盖多条紧密相关的需求，但不能借此引入需求目录中不存在的新产品
承诺。架构约束由架构审查验证，不强行改写成界面场景。

## 加入、授权与身份

### AC-JOIN-001：确认授权前不发送课堂数据

覆盖需求：`REQ-CONSENT-001`、`REQ-CONSENT-002`、`REQ-JOIN-001`

**Given** 学生已经通过插件打开有效加入 URL，并且尚未确认加入

**When** 学生查看课堂信息、输入显示名称并选择工作区

**Then** 插件显示课堂名称、允许的文件类型、同步边界、接收方、持续时间和停止方式

**And** 插件说明未保存文本可能同步、普通源码中的秘密可能无法识别，以及教师可能看到安全化错误信息

**And** 插件不为 CodePulse 读取或发送代码、编辑活动或心跳

### AC-JOIN-002：加入时同步符合规则的活动文件

覆盖需求：`REQ-JOIN-002`、`REQ-SYNC-001`、`REQ-SYNC-003`

**Given** 学生已经选择工作区，活动文件位于该工作区内并通过全部同步检查

**When** 学生明确确认加入且服务端接受加入请求

**Then** 插件显示同步已启用

**And** 插件发送该活动文件的一个完整初始快照

**And** 教师端能够查看服务端接受的该文件快照

### AC-JOIN-003：没有符合规则的活动文件时不批量上传

覆盖需求：`REQ-JOIN-002`、`REQ-SYNC-003`

**Given** 学生已经选择工作区，但确认加入时没有打开符合同步规则的活动文件

**When** 学生成功加入课堂

**Then** 插件不扫描或上传整个工作区

**And** 教师端为该学生显示“暂无已同步代码”

### AC-IDENTITY-001：重名参与者保持可区分

覆盖需求：`REQ-IDENTITY-001`、`REQ-IDENTITY-002`

**Given** 课堂中已经存在一名显示名称为“小明”的学生

**When** 另一名学生也使用“小明”成功加入

**Then** 两名学生获得不同的参与者标识和稳定、可区分的展示标签

**And** 每名学生都能看到自己呈现给教师的展示标签

**And** 其中一名学生离开后，另一名学生的展示标签不改变

### AC-IDENTITY-002：课堂人数上限包括离线和已离开记录

覆盖需求：`REQ-IDENTITY-003`

**Given** 课堂已经存在三十个参与者记录，其中可以包含同步中、离线或已离开的参与者

**When** 第三十一名学生尝试加入

**Then** 系统拒绝加入并显示课堂人数已满

**And** 已离开记录不会为本次课堂释放新的参与者名额

## 文件同步与隐私

### AC-SYNC-001：文本修改产生完整快照

覆盖需求：`REQ-SYNC-003`、`REQ-SYNC-004`、`REQ-OBS-002`

**Given** 学生处于同步中，并且正在编辑一个符合全部同步规则的文件

**When** 学生修改文件内容但尚未保存到磁盘

**Then** 插件在短暂防抖后发送该文件的完整编辑器文本快照和工作区相对路径

**And** 快照不包含学生电脑上的绝对路径或字符级编辑历史

**And** 服务端接受后，教师端显示该文件为当前同步文件

### AC-SYNC-002：查看操作不触发代码快照

覆盖需求：`REQ-SYNC-003`

**Given** 学生已经加入课堂并且没有产生新的文本修改

**When** 学生切换文件、移动光标或选择文本

**Then** 插件不因这些操作发送新的代码快照

### AC-SYNC-003：不允许同步的文件在离开设备前被拒绝

覆盖需求：`REQ-SYNC-001`、`REQ-SYNC-002`、`REQ-SYNC-005`

**Given** 教师只允许同步 `.java` 文件，并且学生已经加入课堂

**When** 学生修改所选工作区内一个不符合固定同步边界或扩展名允许列表的文件

**Then** 插件拒绝发送该文件的路径和内容

**And** 学生端显示工作区相对路径和具体拒绝原因

**And** 教师端最多只显示安全化错误类别和发生时间

**And** 该文本修改可以更新编辑活动时间，但不得使教师得知被拒绝文件的名称、路径或内容

**And** 此次拒绝不影响其他合法文件继续同步

### AC-SYNC-004：服务端拒绝绕过客户端边界的请求

覆盖需求：`REQ-SYNC-006`、`REQ-AUTH-002`

**Given** 服务端已经保存某参与者此前成功同步的合法快照

**When** 服务端收到路径穿越、未允许扩展名、敏感相对路径、过大请求或无权参与者提交的快照请求

**Then** 服务端拒绝该请求且不保存被拒绝内容

**And** 服务端不覆盖此前合法快照，也不把参与者标记为离线

**And** 响应和教师端信息不泄露受保护内容或服务端内部细节

## 更新顺序与连接故障

### AC-ORDER-001：后到的旧版本不能覆盖新版本

覆盖需求：`REQ-RELIABILITY-001`

**Given** 服务端已经接受特定参与者某文件的较高版本快照

**When** 服务端随后收到同一课堂、参与者和文件的较低版本快照

**Then** 服务端保留较高版本快照并拒绝旧版本覆盖当前状态

### AC-ORDER-002：相同版本重试保持幂等

覆盖需求：`REQ-RELIABILITY-001`、`REQ-RELIABILITY-002`

**Given** 服务端已经接受某个版本的文件快照，但插件没有收到确认

**When** 插件使用相同版本重新发送同一快照

**Then** 服务端确认该版本已经接受

**And** 重复请求不再次改变代码状态、版本或活动记录

### AC-CONNECTION-001：心跳超时只表示离线

覆盖需求：`REQ-RELIABILITY-006`、`REQ-OBS-001`、`REQ-OBS-005`

**Given** 学生处于同步中，并且服务端已经接受该学生的代码快照

**When** 服务端连续三十秒没有收到该学生的心跳

**Then** 教师端将该学生标记为离线，而不是已离开或停止编程

**And** 教师仍能查看最后成功同步的代码、最后心跳时间和代码同步时间

**And** 教师端提示现有代码可能已经过期

### AC-CONNECTION-002：断线期间只保留每个文件的最新待发送快照

覆盖需求：`REQ-RELIABILITY-003`、`REQ-RELIABILITY-004`、`REQ-RELIABILITY-005`

**Given** 学生处于同步中，随后网络中断且课堂仍然有效

**When** 学生在断线期间多次修改两个符合规则的文件，然后网络恢复

**Then** 插件在断线期间显示正在重连，且不影响本地编辑、运行和调试

**And** 插件限制自动重试频率并避免所有客户端在同一时刻集中重试

**And** 恢复后，插件为两个文件分别发送断线期间的最新完整快照，而不补发每次编辑

**And** 每个待发送快照只在服务端确认后移除

### AC-RECOVERY-001：插件重启后必须重新确认恢复

覆盖需求：`REQ-RELIABILITY-007`、`REQ-RELIABILITY-008`

**Given** 插件持有尚未结束的本地课堂记录，随后插件进程重启

**When** 插件重新启动并发现该记录

**Then** 插件显示原课堂名称、工作区和同步范围，并要求学生确认是否恢复

**And** 确认前不读取或发送代码、编辑活动或心跳

**And** 学生确认且服务端确认课堂有效后，插件使用原身份恢复，并为新快照分配高于重启前的版本

**And** 本地恢复状态不包含持久化的完整代码快照

### AC-RECOVERY-002：教师服务端重启后不伪造课堂恢复

覆盖需求：`REQ-RELIABILITY-009`

**Given** 第一版教师服务端只在内存中保存正在进行的课堂

**When** 教师服务端进程停止并重新启动

**Then** 教师端显示没有正在进行的课堂

**And** 学生插件在无法连接期间显示正在重连

**And** 插件收到原课堂不可恢复的明确结果后停止重试并显示课堂不可用

**And** 系统不显示或导出重启前已经丢失的课堂数据

## 教师观察与学生离开

### AC-OBS-001：不活跃提醒只依据可观察编辑活动

覆盖需求：`REQ-OBS-003`、`REQ-OBS-004`

**Given** 教师使用默认五分钟不活跃阈值，并且某参与者已有编辑活动

**When** 教师服务端记录的最近编辑活动时间距当前服务端时间超过五分钟

**Then** Dashboard 显示不活跃提醒

**And** 提醒不声称学生已经卡住、离开或停止编程

**And** 代码同步、编辑活动和心跳时间分别显示，不使用学生设备时间作比较

### AC-OBS-002：复制只包含当前显示文件的代码

覆盖需求：`REQ-OBS-006`

**Given** 教师正在查看一名参与者当前同步文件的代码

**When** 教师选择一键复制

**Then** 剪贴板只得到当前显示文件的代码文本，不附加身份、路径或其他文件

**And** 没有可显示代码时复制操作不可用

**And** 剪贴板写入失败时 Dashboard 明确提示失败，不显示虚假的成功状态

### AC-LEAVE-001：主动离开首先在学生本地生效

覆盖需求：`REQ-LEAVE-001`、`REQ-LEAVE-002`、`REQ-LEAVE-003`

**Given** 学生正在同步或重连

**When** 学生主动选择离开课堂

**Then** 插件立即停止读取、发送和自动重连，不等待服务端确认

**And** 插件只尽力发送一次离开通知，然后清除本地课堂记录

**And** 服务端收到通知时显示已离开；通知未送达时只能在心跳超时后显示离线

**And** 教师端保留此前成功同步的代码直到课堂结束

**And** 学生再次加入时必须重新查看授权、选择工作区并确认

## 课堂结束与导出

### AC-END-001：确认结束后拒绝所有后续更新

覆盖需求：`REQ-END-001`、`REQ-END-002`、`REQ-AUTH-001`

**Given** 教师拥有结束课堂的权限并打开结束课堂操作

**When** 教师选择导出并结束或直接结束并删除，并确认操作

**Then** 服务端将课堂标记为已结束并拒绝所有后续学生更新

**And** 仍可联系的插件停止同步并显示课堂已结束

**And** 没有教师权限的请求不能执行结束操作

### AC-EXPORT-001：最终 ZIP 只包含结束时的最新已接受文件

覆盖需求：`REQ-EXPORT-001`、`REQ-EXPORT-002`、`REQ-EXPORT-003`

**Given** 教师选择导出最终快照并结束课堂，且课堂中已有多个学生和多个成功同步文件

**When** 系统生成最终 ZIP 并通过浏览器提供下载

**Then** ZIP 按安全化参与者目录和工作区相对目录结构保存每个文件的最新已接受版本

**And** ZIP 不包含绝对路径、编辑历史、未成功同步文件、终端内容、运行结果或结束后的变化

**And** 教师能够指定 ZIP 文件名，并被告知第一版导出文件不加密

### AC-EXPORT-002：导出未确认保存前保留内存数据

覆盖需求：`REQ-DATA-002`、`REQ-END-002`

**Given** 教师已经选择导出并结束课堂

**When** ZIP 生成失败、下载失败，或者教师尚未确认文件已经保存

**Then** 课堂保持已结束并继续拒绝学生更新

**And** 服务端保留课堂内存数据并允许教师重新生成或下载 ZIP

**And** 系统不得自动把一次下载开始或生成成功当作教师已经保存文件

### AC-END-002：直接结束并删除不生成最终快照

覆盖需求：`REQ-DATA-001`、`REQ-DATA-003`、`REQ-RELIABILITY-010`

**Given** 教师选择直接结束并删除并确认操作

**When** 服务端完成课堂结束处理

**Then** 系统不生成最终 ZIP，并清除代码、参与者状态和活动数据

**And** 系统最多保留不含学生代码、身份、路径和活动记录的课堂终止标记

**And** 终止标记有效期间，原加入凭证只能得到课堂已结束结果

## 初始覆盖说明

这些场景是第一版的初始验收基线，不表示只有这些行为需要测试。实现阶段还需要：

- 为文件大小、人数、路径规范化和凭证输入补充边界值与攻击性输入测试；
- 为平台、部署、内存存储和禁止执行代码等系统约束执行架构审查；
- 在真实或模拟局域网中验证三十名参与者、轮询负载、心跳和重连参数；
- 在具体 API 和错误模型确定后补充协议级测试；
- 当需求改变时，先更新需求目录，再检查受影响场景，避免只修改测试而不修改产品承诺。
