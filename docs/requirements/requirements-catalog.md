# CodePulse 第一版需求目录

## 文档状态与用途

状态：M1 Unit 7 待审草案。

本文档只列出第一版 CodePulse 的核心产品承诺和系统约束。生命周期、文件边界和失败行为文档负责解释完整规则；后续验收标准负责列出成功、失败和边界场景；ADR 和设计文档负责记录具体实现机制。

一条核心需求可以包含实现同一用户或安全承诺所必需的若干条件，但不把每个输入分类、状态分支或错误提示拆成新的顶层需求。草案通过后编号保持稳定；删除需求时保留编号空缺，不重新编号。

## 验证方式

- **验收场景**：通过用户或外部接口可观察的 Given/When/Then 场景验证。
- **边界测试**：使用边界值、非法输入或攻击性输入验证。
- **故障测试**：主动模拟进程终止、请求超时或部分失败，验证恢复和数据保留行为。
- **架构审查**：检查职责、数据流和实现约束，适用于不能只从界面判断的要求。
- **课堂实验**：在接近真实人数和局域网条件下验证。

代表性 Given/When/Then 场景见
[第一版初始验收标准](acceptance-criteria.md)。

## 文件同步与隐私

| 编号 | 核心需求 | 详细来源 | 计划验证 |
| --- | --- | --- | --- |
| REQ-SYNC-001 | 插件只能同步所选工作区内、扩展名允许、非符号链接、非敏感路径、不超过 1 MiB 的普通文本文件；路径穿越、工作区外路径和无法安全确认归属的路径必须拒绝。 | [文件同步边界](file-sync-boundary.md) | 边界测试 |
| REQ-SYNC-002 | 同步采用默认拒绝：文件必须通过全部固定检查才能发送，教师允许列表不能覆盖固定安全规则，后续检查不能重新允许已拒绝文件。 | [文件同步边界](file-sync-boundary.md) | 架构审查、边界测试 |
| REQ-SYNC-003 | 学生确认后，插件只因加入时活动文件或实际文本修改产生快照；未保存文本可以同步，仅切换文件、移动光标或选择文本不得触发快照。 | [文件同步边界](file-sync-boundary.md)、[术语表](glossary.md) | 验收场景 |
| REQ-SYNC-004 | 第一版必须在短暂防抖后发送完整文件快照和工作区相对路径，不发送字符级差异或学生电脑绝对路径。 | [学习计划](../roadmap/project-learning-plan.md)、[文件同步边界](file-sync-boundary.md) | 架构审查、边界测试 |
| REQ-SYNC-005 | 插件本地拒绝文件时不得发送路径或内容；学生看到相对路径和详细原因，教师只看到安全化错误类别和发生时间。 | [文件同步边界](file-sync-boundary.md)、[失败行为](failure-behavior.md) | 验收场景、边界测试 |
| REQ-SYNC-006 | 服务端必须重复执行其能够验证的同步边界；拒绝请求不得保存被拒绝内容、覆盖合法快照或把参与者标记为离线。 | [失败行为](failure-behavior.md) | 架构审查、边界测试 |
| REQ-PRIVACY-001 | CodePulse 不得修改或删除学生本地文件，也不得同步普通终端内容、运行结果或完整编辑历史。 | [产品范围](product-scope.md)、[文件同步边界](file-sync-boundary.md) | 架构审查、验收场景 |

## 更新顺序与故障恢复

| 编号 | 核心需求 | 详细来源 | 计划验证 |
| --- | --- | --- | --- |
| REQ-RELIABILITY-001 | 同一课堂、参与者和文件的快照版本必须单调递增；服务端接受更高版本、幂等确认相同版本，并拒绝较低版本覆盖当前状态。 | [失败行为](failure-behavior.md) | 验收场景、边界测试 |
| REQ-RELIABILITY-002 | 快照请求没有响应时，插件可以使用相同版本重试；重试不得使相同快照重复改变服务端状态。 | [失败行为](failure-behavior.md) | 验收场景、故障测试 |
| REQ-RELIABILITY-003 | 无法连接或没有响应时，插件必须显示正在重连并把课堂状态视为未知；学生本地编辑、运行和调试不受影响。 | [失败行为](failure-behavior.md) | 验收场景、故障测试 |
| REQ-RELIABILITY-004 | 断线期间，插件必须为每个符合规则的已修改文件保留一个最新待发送快照；同一文件的新快照替换旧快照，恢复后发送全部待发送快照，并在确认后移除。 | [失败行为](failure-behavior.md)、[术语表](glossary.md) | 验收场景、故障测试 |
| REQ-RELIABILITY-005 | 自动重试必须限制频率、避免多个客户端同时重试，并在收到终止性结果或学生离开后停止；第一版的具体重试间隔由失败行为和后续设计记录定义。 | [失败行为](failure-behavior.md) | 边界测试、课堂实验 |
| REQ-RELIABILITY-006 | 同步中插件初始每十秒发送心跳；服务端连续三十秒未收到心跳后标记离线，文件拒绝或过期更新不得触发离线。 | [课堂生命周期](classroom-lifecycle.md)、[失败行为](failure-behavior.md) | 验收场景、课堂实验 |
| REQ-RELIABILITY-007 | 插件进程重启后必须重新显示原课堂和同步范围并请求学生确认；确认前不发送数据，确认且课堂有效时恢复原身份，拒绝恢复时按主动离开处理。 | [课堂生命周期](classroom-lifecycle.md)、[失败行为](failure-behavior.md) | 验收场景、故障测试 |
| REQ-RELIABILITY-008 | 同一身份跨插件重启恢复时必须维持快照版本递增和待发送文件恢复；可以持久化版本及相对路径，但不得为恢复目的持久化完整代码快照。 | [失败行为](failure-behavior.md) | 架构审查、故障测试 |
| REQ-RELIABILITY-009 | 教师服务端重启后不得猜测已丢失的内存课堂状态；原插件先保持重连，收到不可恢复结果后停止并提示课堂不可用，教师和学生必须创建并加入新课堂。 | [失败行为](failure-behavior.md) | 验收场景、故障测试 |
| REQ-RELIABILITY-010 | 课堂结束后，服务端保留不含学生代码和身份信息的终止标记，最长二十四小时或直到进程停止；标记存在时原加入凭证只能得到课堂已结束结果。 | [课堂生命周期](classroom-lifecycle.md)、[术语表](glossary.md) | 验收场景、边界测试 |

## 尚未形成需求的开放问题

- 扩展名允许列表的默认值和大小写规则；
- 文本与二进制文件的具体判断算法；
- 已知敏感路径规则的完整清单；
- 同步错误事件的合并和限流；
- 真实课堂使用的局域网传输加密与服务端身份校验机制；
- HTTP 状态码和机器可读错误代码；
- 导出文件名安全化规则和学生目录短标识格式；
- 是否以及何时加入教师服务端本地持久化。
