# ADR-0007：普通 HTTP 仅用于非敏感学习原型

- 状态：已接受，临时限制
- 日期：2026-07-23

## 背景

CodePulse 需要通过局域网把学生代码和 bearer credential 发送到教师服务端。普通 HTTP 不提供传输加密、内容完整性或服务端身份校验，同一网络中的第三方可能读取或修改流量并取得凭证。

局域网 HTTPS 不只是打开一个框架选项。学生插件还必须可靠识别教师服务端，可能需要受信任证书、证书或公钥固定、证书分发与更新、主机名处理和失败提示。项目当前需要先学习 HTTP、Spring MVC 和端到端集成，尚未完成这项安全设计。

教学内容的经济价值可能较低，但威胁还包括好奇的同网用户、错误配置、共享加入信息、恶意本地软件和代码中意外出现的个人信息或秘密。低价值只能降低部分攻击动机，不能提供机密性。

## 决策因素

- 不虚假宣称普通 HTTP 能够保护学生代码和凭证；
- 允许在学习底层 HTTP 时完成受控的端到端原型；
- 阻止尚未保护的原型被误用于真实学生课堂；
- 把传输加密和服务端身份校验作为真实课堂发布门槛；
- 为以后专门学习 TLS、证书固定和网络攻防保留清晰边界；
- 继续满足不租用远程服务器和安装后可离线运行的长期目标。

## 考虑过的方案

### 受控 HTTP 学习原型

开发与集成阶段可以在 loopback 或受控局域网中使用普通 HTTP，并只处理虚构、非敏感测试代码。该方案最适合观察 HTTP 请求和 Spring MVC 行为，但没有网络机密性，必须明确标记为不可用于真实课堂。

### 局域网 HTTPS 与服务端固定

教师服务端可以生成证书，学生通过加入信息获得可信指纹或公钥，并由插件固定预期服务端。该方案能够在不依赖公网服务的情况下提供加密和服务端身份校验，但需要正确设计首次信任、证书生命周期和 Node.js 客户端校验。

### 公网证书或反向隧道

域名、公开受信任证书或 Cloudflare Tunnel 可以简化部分浏览器和客户端信任问题，但会引入互联网、外部服务、域名或隧道配置。它适合后续远程课堂研究，不能满足本地课堂安装后不依赖互联网的全部目标。

### 应用层自行加密

应用可以在 HTTP 正文内自行加密代码和凭证，但仍需解决密钥交换、服务端身份、重放、完整性和算法升级，实际上是在重复构造 TLS 已解决的协议问题。第一版不采用自制加密协议。

## 决策

CodePulse 的 HTTP 端到端实现只作为学习和集成原型，限于 loopback 或明确受控的局域网，并且只能使用虚构、非敏感的测试代码。文档、界面和课堂运行说明不得把该原型描述为适合真实学生课堂。

在处理真实学生代码前，项目必须通过新的 ADR 选择并实现传输加密与教师服务端身份校验。验证至少包括：网络观察者不能读出代码或 bearer credential、插件拒绝身份不匹配的服务端、凭证不会进入普通查询参数或日志，以及失败时不会静默降级到 HTTP。

最低传输保护是发布门槛。更深入的威胁建模、受控抓包、限流、滥用检测、依赖安全检查、模糊测试和攻击模拟属于未来候选学习范围，可以在获得授权的本地实验环境中逐步加入。

本决策不选择最终证书方案，也不承诺普通 HTTP 原型具有网络隐私。ADR-0006 的授权边界仍需在原型中实现和测试，但其 bearer credential 只有在加密传输后才能抵抗网络窃听。

## 影响

### 正面影响

- 项目可以先通过可观察的普通 HTTP 学习 Web 基础和系统集成；
- 安全限制被写成明确发布门槛，不依赖开发者记忆或口头提醒；
- 以后能够在已有系统上独立学习 TLS、服务端身份和网络测试；
- 不需要现在仓促选择容易产生错误安全感的自签名证书流程。

### 负面影响

- 初始集成版本不能用于真实学生课堂；
- 项目在成为课堂可用产品前必然需要额外安全设计和实现工作；
- 测试人员必须避免把真实代码、凭证或个人信息放入 HTTP 原型；
- 如果发布说明或界面没有保留限制提示，原型可能被误用。

## 重新评估条件

准备进行真实课堂试用、处理任何真实学生代码、引入远程课堂或开始分发可安装版本前，必须重新评估并取代本决策。不能以教学代码价值低、网络看似可信或攻击概率小为由跳过发布门槛。

新的 ADR 应比较局域网证书固定、受信任本地证书和远程隧道等方案，并记录证书分发、更新、撤销、失败提示和安全验证方式。
