# ADR-0001：教师服务端采用 Spring Boot 与 Spring MVC

- 状态：已接受
- 日期：2026-07-22

## 背景

CodePulse 的教师服务端需要接收并验证学生插件发送的代码快照、编辑活动和心跳，维护课堂的当前状态，并向教师 Dashboard 提供查询、结束课堂和导出最终快照的能力。学生代码始终在学生电脑本地运行，教师服务端不执行学生代码或命令。

第一版服务端运行在教师电脑上，面向同一局域网内不超过三十个参与者。主要交互是普通 HTTP 请求和 JSON 数据，不要求通过响应式流处理大规模长连接，也不要求服务端集群或远程执行环境。

项目开发者熟悉 Java 基本语法和面向对象编程，能够教授 AP Computer Science A，但尚未使用 Spring 开发完整项目。项目开发者曾使用 Django 开发过小型 Web 应用，因此 Django 是可行方案，而不是因缺乏使用经验被排除。

CodePulse 首先是学习 Spring、TypeScript 和工程过程的项目。尽快交付可运行产品不是唯一目标；技术选择还需要提供新的、能够通过真实项目实践的学习内容。

## 决策因素

- 让项目开发者在已有 Java 基础上系统学习 Spring 和 Java Web 工程实践；
- 满足不超过三十名参与者的普通 HTTP 与 JSON 通信需求；
- 保持第一版请求处理模型直接、可调试且容易测试；
- 避免为了尚不存在的大规模并发需求引入响应式编程复杂度；
- 使用具有成熟文档、测试支持和长期维护生态的 Java Web 方案；
- 接受学习速度优先于最短交付时间。

## 考虑过的方案

### Spring Boot 与 Spring MVC

Spring Boot 负责应用启动、依赖组合、自动配置和内嵌服务器集成；Spring MVC 提供基于请求与响应的 HTTP 路由、参数绑定和 JSON Web 编程模型。两者共同组成候选方案，不是相互替代的框架。

该方案符合学习目标，也足以承担第一版的请求规模。代价是需要学习依赖注入、自动配置、应用生命周期和 Spring 测试方式，并接受高于极简服务器的启动与资源开销。

### Django

Django 能够快速建立常规 Web 应用，项目开发者也已有小型项目经验。选择 Django 可能缩短初始开发时间，但会把学习继续集中在已经接触过的框架上，不能实现本项目学习 Spring 的主要目标。

### Node.js 与 TypeScript

Node.js 可以让插件、Dashboard 和服务端共享 TypeScript 语言与部分数据模型，减少跨语言切换。然而，服务端也使用 TypeScript 会显著减少 Java 与 Spring 的实践范围，不符合当前学习目标。

### 原生 JDK HTTP 或 Servlet

直接使用 JDK HTTP 能力或 Servlet 可以减少框架抽象，并帮助理解 Spring MVC 建立在哪些 Web 基础之上。但将它作为最终服务端需要手工处理更多启动、路由、序列化、错误处理和测试基础设施。这些技术将在 M3 中用于学习底层概念，不作为第一版产品服务端框架。

### Spring WebFlux

Spring WebFlux 提供响应式和非阻塞编程模型，适合确实需要异步数据流或大量并发连接的系统。CodePulse 第一版人数较少，并计划使用普通 HTTP 请求和 Dashboard 轮询；现在引入响应式类型、背压和不同的调试模型只会增加复杂度。

## 决策

CodePulse 第一版教师服务端采用 Spring Boot 建立和运行应用，采用 Spring MVC 实现普通 HTTP 与 JSON 接口。

在正式使用 Spring MVC 前，M3 将通过 JDK HTTP 与 Servlet 小练习学习请求、响应、路由、Servlet 容器和 JSON 序列化，使 Spring 提供的自动化能够与底层概念对应，而不是只依赖框架惯例。

本决策不决定构建工具、Dashboard 更新方式、接口路径、认证机制、课堂状态存储方式或未来是否使用 WebSocket；这些问题分别记录和评估。

## 影响

### 正面影响

- 项目能够在真实需求中学习 Spring Boot、Spring MVC 和 Java Web 工程实践；
- 已有 Java 基础可以降低同时学习新语言和新框架的认知负担；
- 请求与响应模型符合第一版 HTTP 与 JSON 交互，行为较容易跟踪和测试；
- Spring 生态为后续参数校验、安全控制、测试和打包提供成熟能力。

### 负面影响

- 第一版开发速度可能慢于继续使用已有 Django 经验；
- 自动配置和依赖注入可能隐藏对象创建与服务器启动过程，需要专门学习和验证；
- Spring Boot 的启动时间和内存占用高于极简 HTTP 服务端；
- 项目同时包含 Java 和 TypeScript，需要维护两套语言工具链与数据契约。

## 重新评估条件

出现以下情况之一时重新评估本决策，但不因猜测未来需求而提前更换框架：

- 在目标教师电脑上的测量结果表明启动时间或资源占用不可接受；
- 未来经过确认的通信需求发生重大变化，并且 Spring MVC 对实现造成可验证的限制；
- 项目的主要学习目标不再包含 Spring，且维护者技能或维护成本成为更重要的约束；
- Spring Boot 或 Spring MVC 无法继续获得项目所需的安全维护支持。

重新评估时创建新的 ADR，并将本文档标记为被取代，而不是改写当时的决策背景。
