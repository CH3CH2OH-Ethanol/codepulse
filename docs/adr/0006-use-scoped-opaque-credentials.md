# ADR-0006：使用分权的不透明凭证与本机教师会话

- 状态：已接受
- 日期：2026-07-23

## 背景

CodePulse 存在三种权限不同的调用者：尚未加入的学生、已经加入的参与者和教师。加入 URL 会被分享给全班，因此持有加入信息不能自动获得查看全班代码、结束课堂或导出快照的权限。

服务端不能依赖 Dashboard 隐藏按钮、请求中的参与者标识或客户端自觉来实施授权。每个受保护接口都必须根据服务端认可的凭证决定调用者能够执行的操作，并阻止参与者代表其他人提交或读取数据。

第一版没有长期账户、远程身份提供商或跨服务验证需求。课堂与授权状态都只保存在教师服务端内存中，课堂结束后不需要继续验证旧凭证。

## 决策因素

- 将加入、参与者提交和教师管理权限明确隔离；
- 服务端在每次请求上执行授权，而不是信任客户端界面；
- 凭证不可预测、可失效且不会暴露业务数据；
- 不为临时本地课堂引入账户注册、OAuth 或身份提供商；
- 教师权限不能通过分享给学生的加入 URL 获得；
- 凭证不得进入日志、错误响应或普通业务数据。

## 考虑过的方案

### 分权的不透明随机凭证

服务端为不同能力生成密码学安全的随机秘密值，并在内存中记录它们对应的课堂、参与者和权限。凭证本身不包含业务声明，服务端查找当前状态后作出授权决定。

该方案容易失效和限制作用域，也不需要签名或密钥轮换。代价是服务端必须保存凭证状态，并且 bearer credential 一旦泄露，持有者可以在失效前使用对应权限。

### 所有客户端共享一个课堂秘密

教师和学生共享一个秘密实现最简单，但任何学生都会获得读取全班、结束课堂或导出的能力。无法区分参与者也使冒充和单独失效凭证难以处理，因此不满足最小权限原则。

### JSON Web Token

JWT 可以携带签名声明，使服务端不查询会话状态也能验证权限。CodePulse 本来就需要内存课堂状态，并要求课堂结束和参与者离开后立即失效；使用 JWT 仍需处理撤销状态，同时增加签名算法、密钥和声明校验复杂度。

### 用户账户与外部身份提供商

账户、OAuth 或学校身份系统可以提供长期身份和集中登录，但会引入互联网依赖、账户生命周期、个人信息和外部服务配置。第一版只需要临时课堂能力，不需要知道学生的长期真实身份。

### 只依赖本机网络地址

教师请求来自教师电脑是有用的附加边界，但 IP 地址本身不是可靠身份凭证。代理、隧道和网络配置可能改变请求来源，浏览器访问本机服务还需要防止恶意网页借教师浏览器发起请求。

## 决策

CodePulse 使用服务端生成并保存在内存中的不透明随机凭证。加入凭证、参与者凭证和教师会话标识都必须具有足够的密码学随机性，不能由课堂名称、时间、参与者编号或其他可预测数据生成。

加入凭证只允许请求加入对应课堂。成功加入后，服务端为该参与者生成独立参与者凭证；该凭证只能提交对应参与者的快照、编辑活动、心跳和离开通知，不能读取课堂数据或使用其他参与者标识。参与者离开或课堂结束后凭证失效。

教师 Dashboard 只能从教师本机的 loopback（本机回环地址）入口建立教师会话。教师接口同时要求有效教师会话和本机来源；浏览器会话使用服务端管理的 cookie，并在实现阶段配置 `HttpOnly`、`SameSite`、来源校验和适当的 CSRF 防护。教师会话允许读取全班状态与代码、结束课堂和导出，但不能从学生加入 URL 建立。

服务端在每个接口执行授权并核对凭证作用域。请求路径或正文中的课堂与参与者标识不能扩大凭证权限。凭证不记录到日志，不出现在错误响应中；服务端 API 接收凭证时使用请求头、受保护 cookie 或请求正文，而不是普通查询参数。

本决策只提供身份与权限边界。普通 HTTP 无法保护 bearer credential 或学生代码在网络传输中的机密性，学习原型的传输限制由 ADR-0007 记录。

## 影响

### 正面影响

- 分享加入 URL 不会同时分享教师权限；
- 每名参与者只能改变自己的状态，服务端能够独立失效其凭证；
- 教师管理接口与学生局域网访问形成额外的本机边界；
- 不透明凭证不需要 JWT 声明、签名密钥或长期账户系统；
- 课堂结束时清除内存状态即可使相关凭证失效。

### 负面影响

- 服务端必须安全保存和比较多类凭证，并测试每个接口的授权矩阵；
- bearer credential 泄露后可以被冒用，授权分层不能替代传输加密；
- 本机教师会话仍需处理 cookie、CSRF、恶意本地进程和浏览器安全边界；
- 未来远程教师访问或多教师协作需要新的身份与授权设计。

## 重新评估条件

出现长期账户、多教师协作、学校统一登录、远程教师访问或跨服务验证需求时重新评估本决策。届时可以比较服务器会话、外部身份提供商和标准化 token，但不能让新增方案扩大现有最小权限边界。

改变凭证模型时创建新的 ADR，并重新执行所有加入、参与者越权、教师操作、失效和凭证泄露测试。
